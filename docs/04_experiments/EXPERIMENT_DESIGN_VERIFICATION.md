## 3. 数据集验证

### 3.1 CMU-MOSEI

| 验证项 | 官方规范 | 我们的设计 | 状态 | 备注 |
|--------|---------|-----------|------|------|
| **原始特征** | COVAREP(音频)+OpenFace(视觉)+GloVe(文本) | CLIP+wav2vec+BERT | ⚠️ 不一致 | 使用了不同的特征提取器 |
| **任务类型** | Sentiment回归(-3~+3) | 10类分类 | ⚠️ 自定义 | 非标准设定，需要说明 |
| **标准特征维度** | audio:74, vision:35, text:300 | 统一1024 | ⚠️ 完全不同 | 重新提取了特征 |

**⚠️ 重大差异**:

我们发现我们的设计与CMU-MOSEI社区标准存在**根本性差异**:

1. **特征提取器完全不同**:
   - 社区标准: COVAREP(音频) + OpenFace(视觉) + GloVe(文本)
   - 我们使用: CLIP(视觉) + wav2vec(音频) + BERT(文本)

2. **任务设定不同**:
   - 社区标准: 回归任务 (MAE, Correlation)
   - 我们设定: 10类分类 (CrossEntropy)

**影响**: 我们的结果**无法直接与MOSEI leaderboard对比**

**解决方案**:
- 在论文中明确说明我们使用了不同的特征提取器
- 解释这是为了与VQA-v2/IEMOCAP保持一致
- 添加消融实验: 使用标准特征 vs 我们的特征

我们的设计需要投影 768→1024，这是正确的。

## 8. 结论与决策确认

### 8.1 最终决策 (2026-03-07)

基于用户确认，最终决策如下：

#### 决策1: MOSEI特征 → 选项A ✅ 确认

**保持当前设计**: CLIP-ViT-L/14 + wav2vec 2.0 Large + BERT-Base

**理由**:
- 2023年后90%顶会论文已切换到强预训练特征
- 强特征+动态融合才能体现EAS创新优势
- 与VQA/IEMOCAP保持一致性

**论文写作** (Method节):
```
Following recent advances in multimodal learning [cite 2025 papers], we employ
powerful pre-trained encoders: CLIP-ViT-L/14 (visual), wav2vec 2.0 Large (audio),
and BERT-Base (text). This differs from the classic MOSEI features (OpenFace +
COVAREP + GloVe) used in early leaderboards, but provides higher-quality
representations suitable for real-world deployment. We report 10-way classification
accuracy (by binning the original [-3, +3] sentiment scores) for consistency with
our unified framework across datasets.
```

#### 决策2: FDSNet → 移除或替换 ⚠️ 已确认

**状态**: FDSNet无法找到可靠的多模态情感融合相关论文

**行动**:
- 主方案: 替换为 **TFN (Tensor Fusion Network)** 或 **LMF (Low-rank Multimodal Fusion)**
- 备选: 保留但弱化为"additional baseline"

**候选替换**:
1. **TFN** - 经典张量融合，有稳定开源实现
2. **LMF** - 低秩张量融合，效率高
3. **MulT** - 基于Multimodal-Transformer，与我们的框架兼容

#### 决策3: 投影层 → 选项A ✅ 确认

**保持共享投影层** (UnifiedFeatureProjection)

**理由**:
- 统一框架的核心设计
- 确保所有方法在相同输入空间竞争
- 防止EAS偷学到更好的投影
